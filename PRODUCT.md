# Product

## Register

product

## Users

A single user: the developer/maintainer themselves. No accounts, no
collaborators — the app runs locally and the data never leaves the machine
unless explicitly exported. The context is mid-workflow: reaching for a
prompt, skill, tool, or resource while doing other work, needing to find it
and get back to what they were doing in seconds. The job to be done is
retrieval and reuse, not browsing or discovery — get in, get the asset,
copy it, get out.

## Product Purpose

A personal catalog for the prompts, skills, tools, and resources used to get
work done, replacing prompts scattered across notes apps, chat histories,
and text files with one searchable, taxonomy-coded library. Success looks
like: the user stops keeping prompts in scratch files, any asset is
findable in seconds via search/tag/type, copying a prompt (raw or
variable-filled) takes one click, and the whole library can be backed up
and restored from a single JSON file.

## Brand Personality

Minimal and calm — a quiet workspace that gets out of the way rather than
competing for attention while the user is trying to work. Precise and
unshowy: every interaction feels deliberate and immediate, driven at a
comfortable, mouse-first pace rather than a keyboard-first command-palette
feel. Nothing here needs to perform or persuade; it's a personal tool, not
a storefront.

## Anti-references

Generic SaaS-dashboard reflexes: card-grid monotony, gradient text, hero
metrics, tiny uppercase eyebrow labels, decorative glassmorphism. Anything
that reads as a marketing surface or an admin-panel template rather than a
bespoke tool built for one person's actual workflow.

## Design Principles

- **Speed over decoration.** Every screen should get the user to the asset
  they want in seconds, not ask them to admire it.
- **Content is the interface.** Chrome recedes so the prompt/skill/tool text
  itself is the visual focus, not the frame around it.
- **Deliberate, not templated.** Avoid AI-generation reflexes (card grids,
  eyebrows, gradient accents) — this is shaped around one real workflow, not
  a generic dashboard.
- **Consistent primitives.** Feature UI composes the existing token-driven
  design system (Button, Input, Card, Badge, Modal, etc.) rather than
  introducing one-off styling per screen.
- **Local-first trust.** The interface should feel solid and permanent, like
  a personal filing system, not a cloud service that could vanish.

## Accessibility & Inclusion

Standard WCAG AA baseline: sufficient color contrast in both light and dark
themes, visible focus states on every interactive element, semantic markup,
and full keyboard operability. No additional personal accommodations beyond
that baseline at this time.
